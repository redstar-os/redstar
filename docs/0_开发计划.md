## 多架构微内核操作系统完整开发计划

### 支持架构
- **RISC-V 64位** (rv64gc) - 主开发架构
- **x86_64** - 兼容主流PC平台
- **ARMv8-A (aarch64)** - 移动和嵌入式平台

### 第一阶段：内核基础（步骤1-8）

| 步骤 | 任务 | 验证方式 |
|------|------|----------|
| 1 | **编译环境与启动**：Makefile、链接脚本、多架构入口汇编（RISC-V S模式、x86_64 long mode、aarch64 EL1）、QEMU配置 | QEMU启动输出"Hello Kernel" |
| 2 | **串口驱动与日志**：UART16550驱动、printfk实现、内核调试输出 | 控制台显示内核日志 |
| 3 | **进程表结构**：定义`struct proc`、进程状态枚举、寄存器保存区 | 进程表初始化成功 |
| 4 | **上下文切换**：实现`switch_to()`汇编函数、寄存器保存/恢复 | 两个内核进程交替运行 |
| 5 | **调度器**：多优先级队列、enqueue/dequeue、pick_proc算法 | 多进程按优先级调度 |
| 6 | **IPC消息传递**：message结构、SEND/RECEIVE/SENDREC实现 | 进程间消息通信成功 |
| 7 | **时钟中断**：多架构定时器配置（RISC-V CLINT、x86_64 PIT/APIC、aarch64 Generic Timer）、中断处理、时间片轮转 | 时钟中断触发调度 |
| 8 | **系统调用框架**：多架构系统调用处理（RISC-V ecall/sret、x86_64 syscall/sysret、aarch64 svc/eret）、系统调用表 | 用户程序调用内核服务 |

### 启动流程

```
1. QEMU加载内核镜像
2. 内核初始化：
   - 设置页表、异常向量
   - 初始化进程表
   - 启动IDLE、CLOCK、SYSTEM任务
3. 启动核心服务（内核创建第一批用户态进程）：
   - PM（进程管理器）
   - FS（文件系统）
   - TTY（终端驱动）
4. PM初始化：
   - 获取内存信息
   - 设置进程内存映射
5. FS初始化：
   - 挂载ramdisk
   - 初始化文件系统
6. INIT启动（由PM创建）：
   - 执行/etc/rc（可选）
   - 启动Shell
7. Shell运行：
   - 用户交互
   - 执行命令
```

### IPC通信关系

```
用户程序 ←→ PM ←→ 内核(SYSTEM)
    ↓         ↓
   FS ←→ 内核(SYSTEM)
    ↓
  TTY ←→ 内核(SYSTEM)
```

### 关键数据结构

```c
// 进程表
struct proc {
    struct pstackframe p_reg;    // 寄存器保存区
    struct priv *p_priv;         // 权限结构
    int p_nr;                    // 进程号
    int p_rts_flags;             // 运行状态标志
    message *p_messbuf;          // 消息缓冲区
    int p_getfrom;               // 接收来源
    int p_sendto;                // 发送目标
    struct proc *p_nextready;    // 调度队列链接
    // 架构特有字段
    union {
        struct {
            uint64_t p_satp;     // RISC-V页表基址
        } riscv;
        struct {
            uint64_t p_cr3;      // x86_64页表基址
        } x86;
        struct {
            uint64_t p_ttbr0;    // aarch64用户页表基址
            uint64_t p_ttbr1;    // aarch64内核页表基址
        } arm;
    } p_arch;
    uint64_t p_sp;               // 用户栈指针
};

// 消息结构
typedef struct {
    int m_type;                  // 消息类型
    int m_source;                // 发送者
    union {
        struct { int m1_i1, m1_i2, m1_i3; char *m1_p1; } m_m1;
        struct { long m2_l1, m2_l2; int m2_i1, m2_i2; } m_m2;
        struct { char m3_ca1[16]; char m3_ca2[16]; } m_m3;
    } m_u;
} message;
```
